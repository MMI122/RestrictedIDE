# Restricted IDE - Audit Logs

This directory contains audit log files generated by the Restricted IDE.

## Log Format

Logs are stored as newline-delimited JSON (NDJSON) files.

### File Naming
```
restricted-ide-YYYY-MM-DD.log
```

### Log Entry Structure
```json
{
  "timestamp": "2026-01-17T10:30:00.000Z",
  "level": "INFO",
  "module": "PolicyEngine",
  "message": "Action description"
}
```

### Log Levels
- `ERROR` - Critical errors that affect functionality
- `WARN` - Security warnings and blocked actions
- `INFO` - Normal operational events
- `DEBUG` - Detailed debugging information (development only)

## Special Event Types

### Security Events
```json
{
  "type": "SECURITY",
  "event": "ADMIN_LOGIN_FAILED",
  "data": { "attempts": 2 }
}
```

### Audit Events
```json
{
  "type": "AUDIT",
  "action": "FILE_WRITE",
  "details": { "filePath": "/sandbox/test.txt" },
  "success": true
}
```

## Log Rotation

- Logs rotate when file exceeds 10 MB
- Maximum 10 log files retained
- Oldest files deleted automatically

## Reviewing Logs

Use the Admin Panel's Log Viewer, or review files directly:

```powershell
# View recent entries
Get-Content "restricted-ide-2026-01-17.log" -Tail 50 | ConvertFrom-Json

# Filter security events
Get-Content "restricted-ide-2026-01-17.log" | 
  ConvertFrom-Json | 
  Where-Object { $_.message -like "*SECURITY*" }
```

## Privacy

Logs may contain:
- Timestamps of user actions
- File paths accessed
- URLs visited (within whitelist)
- Blocked action attempts

Logs do NOT contain:
- File contents
- Passwords or credentials
- Personal user data
